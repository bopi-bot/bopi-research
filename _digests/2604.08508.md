## problem

whole-body loco-manipulation -- enabling legged robots to manipulate objects larger or heavier than their arm payload capacity. end-to-end RL requires 15+ reward terms and ~2 hours GPU per task, struggles to generalize. end-to-end MPC suffers from curse of dimensionality at 19+ DoF and diverges in single-shooting rollouts of unstable legged dynamics. sumo's key insight: use a pre-trained generalist WBC policy as the low-level controller and steer it online with sample-based MPC. this reverses the typical RL-high/MPC-low hierarchy.

## architecture

two-level hierarchy:

**low-level WBC policy (50 Hz):** pre-trained ReLIC policy for spot (3-legged gait + 4th leg as manipulator), MJLab velocity-tracking for G1 humanoid. takes desired commands, outputs joint-level $u \in \mathbb{R}^{19}$.

**high-level sample-based MPC (20 Hz):** CEM optimizer, 32 parallel rollouts, 1.5s prediction horizon, 4 spline control points, 3 elite samples, noise variance linearly ramped 0.02 -> 0.6.

**action space for spot:** $c\_{\text{policy}} \in \mathbb{R}^{25}$ decomposed into $c\_{\text{base}} \in \mathbb{R}^3$ (SE(2) velocities), $c\_{\text{arm}} \in \mathbb{R}^6$ (arm joint targets), $c\_{\text{gripper}} \in \mathbb{R}^1$, $c\_{\text{leg}} \in \mathbb{R}^{12}$ (4 legs x 3 joints), $c\_{\text{torso}} \in \mathbb{R}^3$.

continuous selection variable $s\_{\text{leg}} \in [-1, 1]$ handles discrete leg choice: FL active when $s < -0.5$, FR active when $s > 0.5$.

**policy-in-the-loop rollouts:** MPC samples compact action vectors, each step goes through the WBC policy before physics simulation. implemented in C++ with thread pool. 32 rollouts take 43.45 ms on intel i7-12700K (within 50ms budget).

**cost functions:**

move task:

$$J\_{\text{Move}} = w\_{\text{goal}} \cdot \|p\_{\text{obj}} - p\_{\text{goal}}\|\_2 + w\_{\text{gripper}} \cdot \|p\_{\text{gripper}} - p\_{\text{obj}}\|\_2 + w\_{\text{vel}} \cdot \|v\_{\text{obj}}\|\_2$$

upright task:

$$J\_{\text{Upright}} = w\_{\text{Upright}} \cdot \|q\_{\text{obj}} - q\_{\text{upright}}\|\_2 + w\_{\text{gripper}} \cdot \|p\_{\text{gripper}} - p\_{\text{obj}}\|\_2$$

## training

| parameter | value |
|-----------|-------|
| CEM rollouts | 32 |
| prediction horizon | 1.5s |
| spline control points | 4 |
| elite samples | 3 |
| update frequency | 20 Hz |
| noise schedule | linear 0.02 -> 0.6 |

low-level policy is pre-existing (ReLIC for spot, MJLab for G1). no training of the WBC in this work. cost weights tuned via bayesian optimization.

## evaluation

**simulation (spot, 20 trials each):**

| object | sumo | E2E MPC | E2E RL |
|--------|------|---------|--------|
| 1.5kg box | ~100% | ~50% | ~100% |
| chair | ~100% | ~30% | ~90% |
| car tire | ~90% | ~50% | ~10% |
| tire rack | ~80% | ~10% | ~0% |
| traffic cone | ~100% | ~30% | ~70% |

sumo uses 3 cost terms vs 15 for E2E RL. generalizes to new objects by swapping the MuJoCo model -- no retraining needed.

**real-world spot (10 trials each):**

| task | weight | time | success |
|------|--------|------|---------|
| tire upright | 15kg | 9.2 +/- 4.7s | 10/10 |
| barrier upright | 16kg | 10.5 +/- 7.1s | 9/10 |
| cone upright | 3.5kg | 10.2 +/- 7.9s | 9/10 |
| chair upright | 16.5kg | 27.3 +/- 19.1s | 8/10 |
| tire stack | 15kg each | 16.5 +/- 8.4s | 8/10 |
| barrier drag | 15kg | 20.2 +/- 6.7s | 9/10 |
| rugged box push | 20kg | 38.3 +/- 16.9s | 10/10 |

**overall: 72/80 (90%) success rate.**

**G1 humanoid simulation:** 37/40 (92.5%) across box push, door open, chair push, table push.

## reproduction guide

1. clone from https://sumo.rai-inst.com/
2. set up MuJoCo simulation (CPU-based, C++ thread pool)
3. obtain pre-trained policies: ReLIC for spot (arXiv:2506.07876), MJLab for G1
4. implement CEM: 32 rollouts, 1.5s horizon, 4 spline points, 3 elites, noise 0.02->0.6, 20Hz
5. define action space as subset of [base, arm, torso, leg, gripper]
6. define cost function (move or upright task)
7. for new tasks: swap MuJoCo object model + adjust cost weights via bayesian optimization
8. spot deployment: fuse joint encoders (333Hz) + MoCap (120Hz), run CEM asynchronously on 64-core desktop, WiFi to robot

## notes

the policy-in-the-loop rollout is the critical technical innovation -- it simultaneously reduces action space dimensionality, stabilizes contact-rich dynamics (avoiding single-shooting divergence), and decouples locomotion from manipulation in cost design. the continuous selection variable for discrete leg choice is clever. order-of-magnitude faster task tuning via bayesian optimization vs RL retraining. 90% real-world success on objects heavier than the robot's arm capacity is impressive. the framework is practical: only requires a pre-trained WBC + MuJoCo model of the object + cost function, no new training data.
