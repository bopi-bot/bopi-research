## problem

unmanned aerial vehicles (UAVs) need to track dynamic targets (vehicles, pedestrians) in urban environments using natural language instructions like "track the royal blue vehicle at a suitable distance." existing approaches either use traditional control (APF, PID) which lack semantic understanding, or general-purpose VLA models ($\pi\_0$, $\pi\_{0.5}$) which have inadequate temporal modeling and a mismatch between high-level semantic features and low-level continuous displacement control. specifically, general VLAs treat all frames equally, wasting compute on redundant historical features, and their action heads are trained for generic manipulation rather than fine-grained 3D spatial tracking.

## architecture

built on top of the open-source $\pi\_{0.5}$ architecture with three modifications:

**backbone**: PaliGemma -- SigLIP vision tower maps single frames to 256 visual tokens; Gemma LLM fuses visual and instruction tokens. proprioceptive state $S\_t = [v\_t, \dot{\psi}\_t]^\top \in \mathbb{R}^4$ (3D linear velocity + yaw rate) is projected separately.

**temporal compression net**: for the three historical frames $I\_{t-3:t-1}$, a linear projection compresses each frame's 256 visual tokens down to 64 tokens. the $64 \times 3$ compressed tokens are concatenated with the current frame's 256 tokens along the temporal dimension, yielding 448 total visual tokens. learnable positional encodings are added to capture inter-frame evolution patterns. this gives the model temporal awareness without the $O(n \times 256)$ token explosion of naive frame stacking.

**dual-branch decoder**: two parallel decoding branches share cross-modal features but are completely decoupled during inference:

1. **spatial-aware auxiliary grounding head**: a compact transformer after Gemma that predicts the target's 3D relative pose $P\_t = [\Delta x\_t^{\text{tar}}, \Delta y\_t^{\text{tar}}, \Delta z\_t^{\text{tar}}, \Delta \psi\_t^{\text{tar}}]^\top \in \mathbb{R}^4$. this branch uses bypass auxiliary design -- it only operates during training via auxiliary supervision loss, backpropagating spatial loss to force the encoder to embed geometric priors into the shared cross-modal tokens. at inference, only the action branch runs.

2. **flow matching action expert**: generates a 25-step continuous action chunk $A\_{t+1:t+25} = \{a\_{t+1}, \ldots, a\_{t+k}\}$ where each action $a\_i = [\Delta x\_i^{\text{act}}, \Delta y\_i^{\text{act}}, \Delta z\_i^{\text{act}}, \Delta \psi\_i^{\text{act}}]^\top \in \mathbb{R}^4$ represents desired 3D relative displacement and yaw change. trained with flow matching using absolute position changes as supervision.

## training

- **hardware**: 2x NVIDIA H100 GPUs
- **batch size**: 64 (global)
- **iterations**: 45,000
- **optimizer**: not specified in main text
- **dataset**: 892,756 frames total -- ~200K expert demonstrations + ~690K automated collection via APF with random perturbation injection for covariate shift mitigation
- **data diversity**: 85 diverse objects, 176 fine-grained tracking tasks, dynamic weather, multiple CARLA towns
- **target kinematics**: vehicle speeds 0--70 m/s, wide UAV yaw dynamics
- **action space**: direct 3D relative displacement + yaw change (not velocity control), k=25 action chunk length
- **loss**: joint loss function combining visual-language alignment and action execution (weights in supplementary)
- **sim-to-real gap**: addressed by APF-based automated data collection with random perturbation injection, teaching the model to recover from off-ideal trajectories

## evaluation

all experiments in CARLA simulator. episodes are 500 steps max; success = surviving 200 steps without losing the target for 15 consecutive frames.

**seen maps** (Town02, 05, 06, 07, 10HD), pedestrian tracking at far distance ($\leq 20$m):

| model | ATF (frames) | SR (%) |
|-------|-------------|--------|
| ACT | 21.52 | 0.00 |
| WALL-OSS | 41.62 | 0.00 |
| $\pi\_0$ | 133.53 | 17.65 |
| $\pi\_{0.5}$ | 214.28 | 44.44 |
| **UAV-Track VLA** | **269.65** | **61.76** |

**zero-shot unseen maps** (Town01, 03, 04), pedestrian tracking at far distance:

| model | ATF | SR (%) |
|-------|-----|--------|
| $\pi\_{0.5}$ | 139.18 | 5.88 |
| **UAV-Track VLA** | **226.90** | **55.00** |

**ablation** (removing auxiliary grounding head, seen maps, far pedestrian):
- w/o aux head: ATF 183.96, SR 39.29%
- with aux head: ATF 269.65, SR 61.76%

**inference latency**: 0.0571s per step, 33.4% faster than base $\pi\_{0.5}$ (0.0857s). the temporal compression net reduces visual tokens from $256 \times 4 = 1024$ to 448, which is where the speedup comes from.

## reproduction guide

1. clone `https://github.com/Hub-Tian/UAV-Track`
2. install CARLA simulator (tested on CARLA 0.9.15)
3. requirements: PaliGemma, PyTorch, CARLA Python API
4. training needs 2x H100 GPUs (64 batch size, 45K iterations)
5. dataset: either collect your own via the provided APF-based pipeline or use the pre-collected dataset from the repo
6. evaluation: run in CARLA with the provided evaluation scripts

**gotchas**: the temporal compression net is a simple linear projection -- not a learned attention-based compressor. the auxiliary grounding head is training-only (bypass design) so inference is single-branch. action space is relative displacement, not velocity, which affects how you integrate actions in a real deployment.

## notes

the key architectural insight is the temporal compression net that reduces historical frame tokens by 4x (256 to 64 per frame). this is a simple but effective solution to the token explosion problem in multi-frame VLA inputs. it's similar in spirit to the unified self-attention memory pattern in video world models, but uses linear projection rather than learned attention for efficiency.

the bypass auxiliary design is notable: the grounding head provides spatial supervision signal during training but is completely removed at inference. this forces the shared encoder to learn geometric priors that the action expert can implicitly use, without any inference cost overhead. this is a clean version of the "zero-init cross-attention preserves pretrained representations" pattern -- the auxiliary head doesn't interfere with the main action pathway at inference.

the 33.4% latency reduction from token compression is significant for real-time UAV control (17.5 Hz effective rate). this connects to the "can fast-slow inference make foundation models real-time" angle -- the temporal compression is essentially a fast-path for historical context.

the model's strong pedestrian tracking performance (61.76% SR) vs weaker vehicle tracking (37.88%) suggests the architecture may be biased toward smaller, more predictable targets. the sim-to-real gap for aerial tracking remains a major open question -- CARLA provides idealized sensing that real drones don't have.
