## problem

VLA models (RT-2, $\pi\_0$, OpenVLA, SmolVLA) generalize well for open-ended manipulation in well-lit lab settings, but their frame-based RGB perception is fragile under sensing-stage degradations: extreme low light, motion blur, and black clipping. increasing exposure to compensate for low light directly trades off against motion blur and inference latency. existing solutions (image enhancement, data augmentation) operate on already-degraded measurements and cannot recover information lost at capture time.

the gap: nobody has integrated event cameras into pretrained VLA architectures for manipulation. event cameras asynchronously capture per-pixel brightness changes with microsecond latency and high dynamic range, naturally mitigating motion blur and maintaining signal fidelity under extreme lighting -- but their sparse, non-stationary streams are poorly matched to image-domain pretraining that modern VLAs rely on.

## architecture

### base model: SmolVLA (~0.5B params)

SmolVLA uses SigLIP vision encoder + SmolVLM backbone + transformer-based action expert. encodes multi-view RGB observations, language instructions, and robot states into unified tokens. action expert generates action chunks via interleaved self/cross-attention. synchronous inference with action chunk size of 40 at 30 Hz (~1.33 s per forward pass).

### event input: DAVIS346

colored DAVIS346 event camera ($346 \times 260$ resolution) rigidly wrist-mounted on SO100 6-DoF arm. provides pixel-aligned asynchronous events and RGB frames with microsecond-resolution timestamps. wide-angle lens (2.2 mm, F/1.8, FoV $> 100°$). fixed hardware gain, 30 FPS, 10 ms default exposure.

### event windowing

raw event stream: $\mathcal{E} = \{e_i\}_{i=1}^{M}$, $e_i = (x_i, y_i, t_i, p_i)$ where $p_i \in \{+1, -1\}$.

**count-based windowing** (optimal): retain the most recent $N$ events up to exposure end time $t_e$:
$$\mathcal{E}_{t_e} = \{e_i \mid t_i \leq t_e\}, \quad \mathcal{W}_I = \{e_k\}_{k=|\mathcal{E}_{t_e}|-N+1}^{|\mathcal{E}_{t_e}|}$$

optimal $N = 2000$ events. time-based windows (5/20/40 ms) are unreliable due to arm motion pauses and speed changes.

### event representation

polarity-agnostic accumulated count frame, demosaiced to 3 channels:
$$\tilde{E}(x,y) = \sum_{e_i \in \mathcal{W}_I} \delta(x - x_i, y - y_i), \quad E = \mathcal{D}(\text{Norm}(\tilde{E}))$$

where $\delta(x,y)$ is the 2D Kronecker delta, $\text{Norm}(\cdot)$ is min-max normalization to $[0,1]$, and $\mathcal{D}(\cdot)$ is Bayer demosaicing. count-based accumulation outperforms voxel grids, time surfaces, and polarity-aware sum representations.

### event fusion strategies

**overlay (parameter-free):** directly replaces RGB pixels with event color at event-triggered positions:
$$I^{\text{o}}(x,y) = \begin{cases} I(x,y), & |\mathcal{E}_{(x,y)}| = 0 \\ \boldsymbol{c}(p_j), & j = \operatorname*{argmax}_i t_i, \; e_i \in \mathcal{E}_{(x,y)} \end{cases}$$

zero additional parameters, plug-and-play.

**hierarchical event adapter (13.31M params, <3% of total):** lightweight ViT-style event encoder (4 transformer blocks, hidden dim 384) with weight-sharing $16 \times 16$ patch embedding. progressive fusion at SigLIP layers 3, 6, 9, 12:
$$E^{(l+1)} = \mathcal{G}_{l+1}(E^{(l)}), \quad F^{(l+1)} = \mathcal{F}_{l+1}(\text{Fuse}(F^{(l)}, E^{(l)}))$$

where $\text{Fuse}(\cdot) = \text{MLP}(\text{Concat}(\cdot))$ with hidden dim 1536, and event features are projected to 768-dim before concatenation.

## training

**dataset:** 724 teleoperated episodes, 339,310 total frames on SO100 + DAVIS346:
- 310 Pick-Place, 244 Sorting, 170 Stacking
- 305 normal illumination (200 lux), 419 low-light (100/75/40 lux)
- 20 recovery episodes with random initial arm poses per task

three tasks: Pick-Place (grasp + place into cup), Sorting (color-discriminated placement), Stacking (precise vertical placement).

**training stages:**

| stage | components | iterations | lr | batch size |
|---|---|---|---|---|
| Action | action expert + projection MLP | 20k | $2 \times 10^{-4}$ | 256 |
| Event | event adapter + fusion (from scratch) | 10k | $5 \times 10^{-4}$ | 128 |
| Joint | adapter + fusion + action expert + projection | 10k | $1 \times 10^{-4}$ | 128 |

VLM backbone frozen in all stages. random image dropout at 50% during event adapter training to prevent shortcut learning on RGB cues. single NVIDIA A800 GPU.

**training recipe ablation (30 lux Pick-Place):**

| strategy | success rate (%) |
|---|---|
| Action only (no event) | -- |
| Action $\rightarrow$ Joint | 75.0 |
| Event $\rightarrow$ Joint | 50.0 |
| Action $\rightarrow$ Event $\rightarrow$ Joint (no sharing) | 80.0 |
| Full recipe (with weight sharing) | **90.0** |

## reproduction guide

1. clone repo: `git clone https://github.com/JJayzee/E-VLA`
2. obtain SO100 6-DoF arm and DAVIS346 event camera with wide-angle lens (2.2 mm, F/1.8)
3. collect teleoperated demonstrations in LeRobot format under multiple illumination levels
4. for overlay fusion: accumulate recent 2000 events into count frames, overlay on RGB using polarity-to-color mapping, feed to standard SmolVLA pipeline
5. for event adapter: train 3-stage pipeline (Action 20k → Event 10k → Joint 10k) on single A800, with 50% image dropout
6. inference on NVIDIA AGX Orin with synchronous mode, action chunk size 40 at 30 Hz

**key design choices:** count-based windowing with $N=2000$, weight-sharing patch embedding in event adapter, progressive training schedule with image dropout.

## notes

the most striking result: a parameter-free overlay of event maps onto RGB images improves Pick-Place from 0% to 60% at 20 lux. this is a zero-cost intervention that requires no architectural changes. the learnable adapter pushes this to 90%, but the overlay alone is already a strong practical takeaway.

count-based windowing ($N=2000$) is critical -- time-based windows fail because arm motion is highly non-stationary (pauses, speed changes across task phases). this is a robotics-specific insight that general event vision papers miss.

event cameras alone sustain Pick-Place down to 4 lux (~80% with adapter), an order of magnitude below where RGB collapses (~25 lux). at 2 lux, sensor noise dominates and everything fails.

the main failure modes are informative: (1) color semantics -- events encode intensity changes, not absolute color, so Sorting degrades under low light despite robust grasping; (2) occlusion -- wrist-mounted camera gets blocked by grasped objects during Stacking, and events don't help since it's a geometric viewpoint problem, not a sensing one.

E2VID reconstruction-based integration performs poorly (30-50%) because VLA manipulation produces irregular, non-stationary event rates that break the temporal assumptions of recurrent reconstruction networks. this is a strong argument against the "reconstruct-then-process" paradigm for event-VLA integration.

dataset: 724 episodes / 339k frames is relatively small but sufficient for the single-arm, 3-task setup. the code + dataset release at the GitHub repo makes this fully reproducible, which is notable for event-based robotics work.
