## problem

LLM RL training is dominated (~70% time) by rollout generation. quantizing rollouts (W4A16) accelerates this but introduces a training-inference mismatch: rollouts sample from a quantized policy while gradient updates use full-precision weights. the mismatch grows with response length as quantization errors accumulate across autoregressive steps, producing "error tokens" -- repetitive, garbled tokens that break trust-region control. QaRL aligns the training forward pass with quantized rollouts and introduces TBPO (trust band policy optimization) to handle error tokens via sequence-level rejection.

## architecture

**QaRL (rollout-aligned quantization-aware RL):**

1. quantized inference engine (vLLM) deploys W4A16 model for rollouts
2. training side maintains BF16 master weights $\theta\_{\text{BF16}}$. during forward pass, quantize on-the-fly: $\hat{W}\_{\text{low-bit}} = \text{Quant}(\theta\_{\text{BF16}})$, execute actual low-bit GEMM (not fake-quant). backward pass uses STE.
3. publish materialized low-bit tensors directly to inference engine, avoiding redundant re-quantization.

**TBPO (trust band policy optimization):** replaces token-level PPO clipping with:

**dual clipping for negative samples:** for $\hat{A} \geq 0$, standard PPO bound $[0, 1+\epsilon\_h]$. for $\hat{A} < 0$, dual-sided band $[1-\delta\_\ell, 1+\delta\_h]$ constraining both bounds. this handles error tokens that concentrate in low-probability regions where the importance ratio is highly sensitive.

**sequence-level objectives:** entire response treated as single action. sequence-level probability is geometric mean of token probabilities:

$$\pi(o \mid q) = \exp\left(\frac{1}{L} \sum \log \pi(o\_t \mid q, o\_{<t})\right)$$

mismatch weight: $\tilde{w}(\theta) = \text{clip}(w(\theta), [-\log c, \log c])$ with cap $c=2$. entire responses exceeding trust band are masked out from policy updates.

$$J(\theta) = \mathbb{E}[\tilde{w}(\theta) \cdot \tilde{r}(\theta) \cdot \hat{A}]$$

## training

| parameter | value |
|-----------|-------|
| framework | Verl (training) + vLLM (inference) |
| hardware | 8x NVIDIA H800 |
| batch size | 512 |
| max prompt/response length | 2048 / 16384 |
| rollout samples (n) | 8 per prompt |
| temperature | 1.0 (rollout), 0.6 (eval) |
| mini batch size | 64 |
| optimizer | **Muon** |
| learning rate | 1e-6 |
| weight decay | 0.01 |
| KL loss | disabled |

**TBPO hyperparameters:** $\epsilon\_h = 0.0004$, $\delta\_h = 0.0007$, $\delta\_\ell = 0.0003$, TIS cap $c = 2$.

**data:** OpenR1-Math-46K (46,000 math problems).

## evaluation

**Qwen2.5-7B-Math (representative):**

| method | AIME 24 | AIME 25 | avg math | avg OOD |
|--------|---------|---------|----------|---------|
| BF16 GRPO | 19.5 | 11.6 | 43.3 | 44.8 |
| W4A16 rollout GRPO | 19.1 | 9.6 | 40.9 | 38.5 |
| **W4A16 QaRL TBPO** | **19.5** | **13.3** | **43.5** | **44.4** |

**Qwen3-30B-A3B-Base (MoE):**

| method | AIME 24 | AIME 25 | avg math | avg OOD |
|--------|---------|---------|----------|---------|
| BF16 GSPO | 27.9 | 21.6 | 52.1 | 67.8 |
| W4A16 rollout GSPO | 22.0 | 18.7 | 45.7 | 61.2 |
| **W4A16 QaRL TBPO** | **27.5** | **22.0** | **51.2** | **67.05** |

on MoE: QaRL recovers +5.5 points avg math over quantized rollout, closing the gap with BF16 (52.1). speedup: 1.3x over BF16. convergence is insensitive to specific low-bit format (W4A16, W8A8, FP8W8A8 converge similarly under TBPO).

## reproduction guide

1. frameworks: Verl for training, vLLM for inference
2. base models: Qwen2.5-Math-1.5B/7B, Qwen3-8B-Base, Qwen3-30B-A3B-Base
3. data: OpenR1-Math-46K
4. hardware: 8x H800 GPUs (single node)
5. implement low-bit GEMM in training forward (not fake-quant), STE for backward
6. publish quantized weights directly to inference engine
7. implement sequence-level ratio (geometric mean of log-ratios)
8. apply dual clipping + TIS cap + sequence-level masking
9. use Muon optimizer (critical for convergence speed)

no code released yet. complete hyperparameters in Appendix B, Table 2.

## notes

the finding that quantization format (W4A16 vs W8A8 vs FP8) doesn't matter once QaRL aligns training with rollouts is striking -- it means the mismatch, not the bit width, is the dominant problem. the "error token" failure mode is specific to quantized long-form generation and requires sequence-level rejection, not token-level clipping. this has practical implications for anyone doing RL post-training on resource-constrained hardware: quantized rollouts are viable if you can align the training computation. Muon optimizer is significantly faster to converge than AdamW for this setup. the 1.3x speedup on 30B MoE with near-BF16 accuracy is immediately useful.
