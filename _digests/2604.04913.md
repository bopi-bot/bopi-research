## problem

Video world modeling requires anticipating diverse future states from past observations. This is critical for autonomous driving and robotics, where multiple plausible futures must be generated for safe planning.

**Discriminative world models** such as DINO-world [3] (Zhou et al., 2025), DINO-Foresight [35], and DINO-WM [88] predict future frames in VFM feature space. They are efficient but produce a single deterministic prediction that collapses to the conditional mean under uncertainty [71], failing to capture the distribution of plausible futures.

**Existing generative world models** such as Cosmos (NVIDIA, 2025) [1], UniSim [8], Joint-Embed Predictive Architecture (JEPA) [12], and discrete autoregressive approaches [34, 10] can generate diverse futures, but suffer from three fundamental inefficiencies:

1. **Pixel-reconstruction spaces** — models like Cosmos optimize latent spaces for pixel-level fidelity (e.g., using continuous or discrete VAE tokenizers trained with MSE/LPIPS losses), modeling fine-grained visual details irrelevant to downstream tasks like segmentation or depth estimation.
2. **Multi-step generation** — diffusion models (e.g., Cosmos uses iterative denoising) and autoregressive models (e.g., LlamaGen [62], VAR [64]) require multiple sequential forward passes per sample, making inference slow.
3. **Spatial redundancy** — representing each frame as a dense grid of spatial tokens creates long, redundant sequences. At $512 \times 512$ resolution with a $16 \times 16$ patch size, each frame yields $32 \times 32 = 1024$ spatial tokens, multiplied by temporal context length.

The authors demonstrate that Cosmos-12B uses ~64,000 TFLOPs and Cosmos-4B uses ~60,000 TFLOPs for a single rollout, while producing futures that align less well with real outcomes than a much simpler discriminative model.

**Key insight**: Predicting in VFM feature space (e.g., DINOv3 ViT-B features) requires far fewer parameters than pixel-reconstruction approaches, but existing VFM-based methods remain discriminative. This paper bridges the gap by making VFM-based prediction *generative* while keeping it extremely efficient.

## architecture

### Vision Foundation Model (VFM)

A frozen **DINOv3 ViT-B** [60] (Siméoni et al., 2025) backbone with patch size $16 \times 16$ and feature dimension $D = 768$. Each frame $v\_i \in \mathbb{R}^{H' \times W' \times 3}$ is encoded into patch tokens $x\_i = \phi(v\_i) \in \mathbb{R}^{H \times W \times D}$, yielding $H \times W$ spatial tokens per frame (e.g., $32 \times 32 = 1024$ tokens at $512 \times 512$ input).

### DeltaTok Tokenizer

A **continuous autoencoder** (not VAE) that compresses the VFM feature difference between consecutive frames into a single continuous "delta" token.

**Encoder**: ViT-B architecture (12 Transformer blocks, 12 heads, hidden dim 768), reusing DINOv3 Transformer block implementation with 2D RoPE. Operates on concatenated patch tokens from two consecutive frames: $[x\_t, x\_{t+1}] \in \mathbb{R}^{2HW \times D}$. A learned per-frame embedding distinguishes previous-frame from current-frame tokens. The encoder outputs a single delta token $\Delta x\_{t+1} = g(x\_t, x\_{t+1}) \in \mathbb{R}^{D}$, effectively a 1024× token reduction at $512 \times 512$.

**Decoder**: Same ViT-B architecture (omitting final layer normalization), takes the delta token and the previous frame's patch tokens $x\_t$ to reconstruct the next frame's features: $\hat{x}\_{t+1} = d(x\_t, \Delta x\_{t+1}) \in \mathbb{R}^{H \times W \times D}$.

**Tokenizer loss**: Mean squared error (MSE) between reconstructed and ground-truth VFM features:
$$\mathcal{L}\_{\text{tok}} = \|x\_{t+1} - \hat{x}\_{t+1}\|^2\_2$$

**Initialization**: Truncated normal ($\sigma = 0.02$) for all linear/embedding weights, zero biases, Layer Scale [65] initialized to $10^{-5}$.

### DeltaTok Encoder Variants

The encoder has two modes:

- **Delta mode** (primary): Encodes the *difference* between consecutive frame features into a single delta token. Exploits the fact that consecutive frames mostly share static content.
- **Absolute mode** (fallback): When temporal change is large (low temporal redundancy), the encoder can encode the current frame's features directly as a single absolute token.

### DeltaWorld Predictor

A ViT-B Transformer (12 layers, 12 heads, 768 dim, ~86M parameters) operating on a **one-dimensional temporal sequence** of delta tokens — no spatial dimensions.

**Architecture**: Standard causal Transformer with 1D RoPE (rotating first 60 dimensions per head, leaving final 4 unrotated). Because each frame is a single token, neither block-causal attention masks nor 3D RoPE are needed. The model predicts delta tokens autoregressively.

**Prediction**: Given context delta tokens $\Delta x\_{2:t}$ and noise queries, predicts the next delta token:
$$\Delta \hat{x}\_{t+1}^k = f(q^k, \Delta x\_{2:t}, T\_{1:t}, \tau\_{t+1}) \in \mathbb{R}^{D}$$

**Noise injection**: Queries sampled from $q^k \sim \mathcal{N}(0, 0.02^2 I)$ for $k = 1, \dots, K$.

### Best-of-Many (BoM) Training Objective

To make the predictor generative while maintaining single-pass efficiency:

1. Draw $K$ noise queries $q^k \sim \mathcal{N}(0, 0.02^2 I)$
2. Generate $K$ predictions per spatial location
3. Select the best prediction (lowest loss to ground truth):
$$k^\star = \arg\min\_k \sum\_{h,w} \ell(x\_{t+1,h,w}, \hat{x}\_{t+1,h,w}^k)$$
4. Only supervise the best prediction:
$$\mathcal{L}\_{\text{BoM}} = \sum\_{h,w} \ell(x\_{t+1,h,w}, \hat{x}\_{t+1,h,w}^{k^\star})$$

where $\ell$ is the smooth $\mathcal{L}\_1$ loss with $\beta = 0.1$.

At inference, different noise queries produce different futures in a single forward pass — no iterative denoising or autoregressive decoding over spatial tokens required.

### Autoregressive Rollout

For multi-step prediction, the generated delta token is decoded back to VFM feature space, re-encoded into a delta token (conditioned on the decoded features), and used as context for the next step. This enables arbitrary-length rollouts.

## training

### Tokenizer Training

- **Optimizer**: AdamW with weight decay $10^{-4}$
- **Learning rate**: $10^{-3}$ with linear warmup over 5K steps, then constant
- **Batch size**: 1,024 frame pairs
- **Iterations**: 50K
- **Loss**: MSE between reconstructed and ground-truth VFM features
- **Gradient clipping**: Norm clipping at $10^{-2}$
- **Augmentation**: Random resized crops, scale 0.6–1.0, aspect ratio 3:4–4:3

### Predictor Training (DINO-world and DeltaWorld)

- **Optimizer**: AdamW with weight decay $4 \times 10^{-1}$
- **Learning rate**: $10^{-4}$ with linear warmup over 5K steps, then constant
- **Batch size**: 1,024
- **Training sequence length**: 8 frames
- **Iterations**: 300K (main results), 100K (ablations)
- **Loss**: Smooth $\mathcal{L}\_1$ ($\beta = 0.1$)
- **Gradient clipping**: None
- **Fine-tuning**: 5K iterations at $10\times$ lower learning rate ($10^{-5}$)
- **BoM samples**: $K = 16$ during training

### Data

- ~4M video samples, mostly at $640 \times 360$ resolution, 11 seconds duration, 16 FPS
- Videos span diverse domains (driving, general video), similar in spirit to the DINO-world corpus (~66M samples, 5–60s, 10–60 FPS, not publicly released)
- Temporal offsets $\Delta\tau$ sampled uniformly from $[1/25, 1/3]$ seconds

### Hardware

- Training measured on a single node with **8 NVIDIA H200 GPUs**
- BF16 mixed precision with `torch.compile` (default mode)

### Training Efficiency

| Step | Description | Time (relative) | Memory (relative) |
|------|-------------|-----------------|-------------------|
| (0) Discriminative baseline | DINO-world | 1.0× | 1.0× |
| (1) + BoM training | $K=16$ candidates | ~1.0× | ~1.0× |
| (2) + Frame compression | 1 token/frame | 0.2× | 0.1× |
| (3) + Delta compression | DeltaTok + DeltaWorld | 0.3× | 0.2× |

BoM training adds negligible overhead because candidate selection uses detached parameters (no backprop through most forward passes), and only the best candidate is re-run with gradients.

## evaluation

### Datasets and Tasks

| Dataset | Task | Metric | Split |
|---------|------|--------|-------|
| VSPW [47] | Semantic segmentation (124 classes) | mIoU | Validation |
| Cityscapes [15] | Semantic segmentation (19 classes) | mIoU | Validation |
| KITTI [24] | Monocular depth estimation | $\delta\_1$ | Eigen test |

### Evaluation Protocol

- **Input resolution**: 512×512 (main), 256×256 (ablations)
- **Context**: 4 frames for short-term, 4 frames + autoregressive rollout for mid-term
- **Short horizon**: Direct single-step prediction (~0.2s ahead for VSPW/KITTI, ~0.19s for Cityscapes)
- **Mid horizon**: 3-step autoregressive rollout (~0.6s for VSPW/KITTI, ~0.56s for Cityscapes)
- **Generative models**: 20 independent rollouts, report "best" (closest to GT in feature space) and "mean" (average features then apply task head once)
- **FLOPs**: Computed for square inputs, doubled (two crops), excluding tokenizer and KV pre-filling for Cosmos

### Key Findings

1. **Best predictions**: DeltaWorld's best-of-20 consistently beats Cosmos-12B across all tasks and horizons, despite 35× fewer parameters and 2,000× fewer FLOPs.
2. **Mean predictions**: DeltaWorld's mean-of-20 is competitive with discriminative DINO-world (within ~1 mIoU), confirming that sampled futures are realistic and well-calibrated.
3. **Cosmos**: Despite its massive scale (4B–12B parameters), Cosmos underperforms even the discriminative DINO-world on several metrics, and its mean/best samples are very similar, indicating limited diversity.
4. **FLOPs**: DeltaWorld uses ~31 GFLOPs vs. ~60K–64K GFLOPs for Cosmos — a ~2,000× reduction.

### Ablation Results (256×256, 100K iterations)

Stepwise improvement from discriminative to generative to delta-compressed:

| Step | Setting | VSPW mIoU | Cityscapes mIoU | KITTI $\delta\_1$ |
|------|---------|-----------|-----------------|-------------------|
| (0) Discriminative baseline | DINO-world | 44.8 | 45.4 | 89.6 |
| (1) + BoM ($K=16$) | Generative, spatial | 43.8 | 43.8 | 89.0 |
| (2) + Frame compression | 1 token/frame | 45.0 | 45.9 | 89.6 |
| (3) + Delta compression | DeltaWorld | **46.1** | **47.1** | **89.8** |

Notable: Delta compression (step 3) actually *improves* accuracy over the discriminative baseline, attributed to the natural prior of the delta formulation — predicting zero change preserves the previous frame, so the model only needs to learn what changes.

### BoM Sample Scaling

With $K = 16$ training samples, DeltaWorld achieves strong best-of-20 performance. Increasing $K$ during training further improves diversity but with diminishing returns.

### Cross-Architecture Validation (Discriminative)

Delta tokens also transfer to DINO-Foresight [35] (a different discriminative architecture), matching performance with 2,048× fewer tokens (5 tokens vs. 10,240 tokens), confirming the generality of the delta compression approach.

## reproduction guide

### Prerequisites

```bash
# Python 3.10+, PyTorch 2.x with CUDA
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
pip install transformers timm
# Clone the repo
git clone https://github.com/amazon-far/deltatok
cd deltatok
pip install -e .
```

### Step 1: Prepare DINOv3 Backbone

```bash
# DINOv3 ViT-B weights (frozen, not trained)
# The authors use DINOv3 from Hugging Face
python -c "from transformers import AutoModel; AutoModel.from_pretrained('facebook/dinov3-vitb14')"
```

### Step 2: Train DeltaTok Tokenizer

```bash
# 50K iterations, batch size 1024, 8x H200
python train_tokenizer.py \
  --backbone facebook/dinov3-vitb14 \
  --lr 1e-3 \
  --warmup_steps 5000 \
  --weight_decay 1e-4 \
  --batch_size 1024 \
  --iterations 50000 \
  --grad_clip 0.01 \
  --loss mse
```

### Step 3: Train DeltaWorld Predictor

```bash
# 300K iterations, batch size 1024, BoM K=16
python train_predictor.py \
  --tokenizer_path checkpoints/deltatok \
  --backbone facebook/dinov3-vitb14 \
  --lr 1e-4 \
  --warmup_steps 5000 \
  --weight_decay 0.4 \
  --batch_size 1024 \
  --iterations 300000 \
  --bom_k 16 \
  --noise_std 0.02 \
  --seq_length 8 \
  --loss smooth_l1 \
  --smooth_l1_beta 0.1
```

### Step 4: Fine-tune

```bash
# 5K iterations at 10x lower lr
python train_predictor.py \
  --resume checkpoints/deltaworld \
  --lr 1e-5 \
  --iterations 5000
```

### Step 5: Evaluate

```bash
# Dense forecasting evaluation (VSPW, Cityscapes, KITTI)
python evaluate.py \
  --checkpoint checkpoints/deltaworld \
  --datasets vspw cityscapes kitti \
  --resolution 512 \
  --num_rollouts 20 \
  --context_frames 4 \
  --horizon short,mid
```

### Gotchas

1. **No official DINO-world code**: The authors re-implemented DINO-world from scratch since no official codebase was released. Their reimplementation numbers may differ slightly from the original paper.
2. **Cosmos evaluation constraints**: Cosmos requires a fixed 9-frame context and 24-frame rollout in a single pass. Its input format requires padding to 640×1024, making direct comparison non-trivial. FLOPs calculations for Cosmos exclude tokenizer and KV pre-filling.
3. **Tokenizer is not a VAE**: It's a plain autoencoder — no KL divergence term, no variational posterior. This means delta tokens are continuous vectors, not samples from a learned distribution.
4. **Training data**: The authors use ~4M samples from an internal video corpus, not publicly available. DINO-world's training data (~66M samples) is also not released. For reproduction, you would need to curate a similar diverse video dataset.
5. **Crop evaluation**: Evaluation uses two potentially overlapping square crops from resized frames, with labels split at the horizontal midpoint. This preprocessing is important for fair comparison.
6. **Delta token fallback**: When temporal change is very large, DeltaTok can fall back to absolute compression (encoding the full frame rather than the difference). The paper does not detail the exact switching criterion.

## notes

### Key Takeaways

1. **Delta tokens are surprisingly effective**: Compressing an entire $512 \times 512$ frame's VFM features into a single 768-dimensional vector loses very little information for prediction purposes. The natural "no-change" prior (zero delta = previous frame) acts as a strong regularizer.

2. **VFM feature space is the right level of abstraction for world modeling**: Operating in DINOv3 feature space rather than pixel/latent space eliminates the need to model irrelevant visual details, enabling a ~300M parameter model to outperform a 12B parameter diffusion model on dense forecasting.

3. **Best-of-Many is a practical generative training scheme**: BoM avoids the multi-step inference cost of diffusion while producing genuinely diverse futures. The key insight is that different noise queries map to different futures, and only the best needs gradient computation.

4. **Efficiency gains are dramatic**: 2,000× fewer FLOPs than Cosmos means DeltaWorld could run in real-time on edge devices — critical for autonomous driving. Training is also 5× faster and uses 5× less memory per iteration.

### Connections to Other Work

- **VFM-based prediction** [3, 88, 35, 70]: DeltaWorld extends this line from discriminative to generative while maintaining efficiency.
- **Cosmos [1] / UniSim [8]**: Large-scale generative world models in pixel space. DeltaWorld demonstrates that VFM feature space is a more efficient target.
- **Video tokenization** [40, 22]: Li et al. (2025) also explore 1D temporal video tokenization but for pixel reconstruction. DeltaTok is unique in targeting VFM feature space and encoding frame *differences* rather than absolute frames.
- **Motion residuals / optical flow** [73, 63]: DeltaTok shares the spirit of encoding temporal change (like H.264 motion residuals or RAFT optical flow) but operates non-spatially as a single semantic token rather than per-pixel vectors, naturally handling occlusions.
- **Representation Autoencoders [87]**: Used for RGB visualization of DeltaWorld predictions (decoding DINOv3 features back to pixels).
- **Joint-Embed Predictive Architecture (JEPA) [12]**: Also operates in VFM feature space but remains multi-step. DeltaWorld shows single-step generation is viable.

### Next-Step Ideas

1. **Action conditioning**: The paper notes that similar noise queries may produce similar futures across scenes, suggesting the query space could serve as implicit action conditioning. Explicit action-conditional generation would enable planning applications.

2. **Scalability**: The predictor is only ViT-B (~86M params). Scaling to ViT-L or ViT-G could further improve accuracy, and the 1D token sequence makes this very cheap. A ViT-L predictor would still be < 1B total params.

3. **Long-horizon rollouts**: Error accumulation during multi-step autoregressive rollouts is acknowledged as a limitation. Approaches like iterative refinement, scheduled noise injection, or diffusion-based post-processing could help.

4. **Distributional guarantees**: BoM lacks an explicit distributional objective (unlike diffusion). Adding a diversity-promoting regularizer or connecting to flow matching could improve coverage of the predictive distribution.

5. **Multi-modal delta tokens**: Extending delta tokens to multiple VFM feature spaces simultaneously (e.g., DINOv3 + depth features) could capture complementary temporal dynamics.

6. **Real-time deployment**: With ~31 GFLOPs and 0.3B params, DeltaWorld is already suitable for edge deployment. Optimizing the DeltaTok decoder (which dominates per-sample FLOPs) through distillation or caching could further reduce latency.
