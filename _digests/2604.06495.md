## problem
SAEs are prone to feature absorption: general features get subsumed by more specific co-occurring features, degrading interpretability despite high reconstruction fidelity. recent work shows SAEs are also brittle out-of-distribution -- features learned on one data distribution don't transfer. sparsity alone is an imperfect proxy for interpretability, and current training objectives don't prevent these failure modes.

## architecture
standard SAE: encoder $e(\mathbf{x}) = \sigma(W\_e \mathbf{x} + \mathbf{b}\_e) \in \mathbb{R}^m$, decoder $d(\mathbf{z}) = W\_d \mathbf{z} + \mathbf{b}\_d \in \mathbb{R}^D$. uses MatryoshkaBatchTopK architecture (state-of-the-art across interpretability benchmarks). dictionary size 4096 on residual stream activations from layer 8 of Pythia-160M and layer 12 of Gemma-2-2B.

the key contribution is a **masked regularization** strategy that breaks co-occurrence patterns during training. rather than token dropout (which removes information), random spans of the input sequence are replaced with a neutral mask string ("..."). this forces the SAE to learn features that activate independently of their typical co-occurring context, preventing feature absorption.

## training
500M tokens from Pile-CC-deduplicated. trained on Pythia-160M-deduped and Gemma-2-2B. uses the dictionary\_learning codebase. all experiments share the same hyperparameters from the base MatryoshkaBatchTopK setup.

## evaluation
improves performance across standard interpretability metrics (reconstruction fidelity, sparsity, downstream task scores) on both Pythia-160M and Gemma-2-2B. most notably improves OOD robustness -- the key problem the paper identifies. features learned with masked regularization transfer better across distributions.

## reproduction guide
1. clone dictionary\_learning repo (github.com/saprmarks/dictionary\_learning)
2. implement masked regularization: replace random input spans with "..." during training
3. train with standard MatryoshkaBatchTopK config, 500M tokens, batch size and LR from base code
4. evaluate with standard interpretability benchmarks
5. compare OOD performance with and without masked regularization

## notes
the masked regularization approach is simple and doesn't require architectural changes -- it's a data-level intervention. the key insight: feature absorption happens because the SAE sees correlated features always co-occurring in training data, so it learns to route the general feature through the specific one. masking breaks this correlation. the "..." replacement string is notably different from token dropout because it preserves the positional structure of the sequence. this connects to the broader question of how to train SAEs that learn causal features rather than correlational ones. the OOD robustness finding is particularly important for mechanistic interpretability in practice: if SAE features don't transfer across distributions, they're limited to analyzing specific deployment contexts.
