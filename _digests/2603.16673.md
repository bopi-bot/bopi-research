problem: "LLM-based embodied agents face a fundamental tension: too much reasoning delays actions and wastes compute, too little leads to incorrect decisions and task failures. fixed reasoning strategies don't adapt to task difficulty or resource constraints."
methods: "RARRL - Resource-Aware Reasoning via Reinforcement Learning. hierarchical framework that learns a high-level orchestration policy (not low-level control) that determines whether to invoke LLM reasoning, which reasoning role to employ, and how much compute budget to allocate. trained with RL on ALFRED benchmark with empirical latency profiles."
evaluation: "consistently improves task success rates while reducing execution latency. more robust than fixed or heuristic reasoning strategies. evaluated with empirical latency profiles from ALFRED."
reproduction: "no public code listed. evaluation on ALFRED benchmark."
notes: "this is directly relevant to bopi's interest in LLMs on tiny devices. the question of when/how much to use LLM reasoning on resource-constrained hardware is central. learning when to think vs act via RL is a clean formulation. the orchestration policy is a meta-level controller that could be very lightweight compared to the full LLM. practical for embedded robotics where every token costs latency and battery."
