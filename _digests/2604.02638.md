## Problem

KV cache quantization reduces storage for LLM inference, but every existing approach requires **dequantization** at query time before computing attention scores. For an 8B-parameter model at context length $T{=}4096$, the KV cache stores 512 MB in FP16. Quantizing to 2--4 bits cuts storage, but the quantized keys must be dequantized $T$ times per query to compute $\mathbf{q}^{\top}\mathbf{k}$, then multiplied --- a dequant $\to$ matmul pipeline that dominates energy and latency on edge devices.

Standard approaches like GPTQ, AWQ, and bitsandbytes all follow this pattern: quantize for storage savings, but pay the cost of dequantization at every attention computation. The paper asks a more fundamental question: **can the SRAM that stores the quantized cache also compute attention scores, eliminating dequantization entirely?**

## Architecture

### Key Insight: Fixed Codebook via Orthogonal Transforms

For a unit-norm vector $\hat{\mathbf{x}} \in \mathbb{R}^{d}$ and an orthogonal matrix $R$, each coordinate of the rotated vector $\mathbf{y} = R\hat{\mathbf{x}}$ follows approximately $\mathcal{N}(0, 1/d)$ for $d \geq 64$ (concentration of measure). The optimal Lloyd-Max quantizer for this Gaussian distribution yields centroids and boundaries that depend **only on $(d, b)$** --- the dimension and bit-width --- not on the input data. This means the codebook is a **design-time constant**.

For $b{=}3$, $d{=}128$: the fixed codebook has 8 centroids + 7 boundaries = 15 FP16 values = **30 bytes**, verified to satisfy Lloyd-Max optimality to machine precision ($< 10^{-11}$).

### Asymmetric Path: Transform on Write, Table-Lookup on Read

The critical architectural insight exploits the inner-product preservation of orthogonal transforms:

$$\langle \mathbf{q}, \hat{\mathbf{k}} \rangle = \langle R\mathbf{q},\; \mathrm{codebook}[\mathrm{idx}] \rangle \times \|\mathbf{k}\|$$

Instead of inverse-transforming each stored key ($T$ times, as in TurboQuant), AXELRAM **rotates the query once** and computes attention entirely in the rotated domain. This is the "asymmetric path":

- **Write path**: Apply randomized Hadamard transform $R = H\_{d} \cdot \mathrm{diag}(\mathbf{s})$ to incoming key, quantize against fixed codebook, store 3-bit indices + FP16 norm in SRAM. No multipliers needed (FWHT butterfly uses only add/sub).
- **Read path**: Rotate query once with same FWHT, pre-compute a lookup table of size $d \times 2^{b}$ (1024 entries for $d{=}128, b{=}3$), then for each key: look up $d$ table values from SRAM indices, accumulate via adder tree, multiply by norm. **No inverse transform circuit exists in the read path.**

### SRAM Macro Design

The smart SRAM macro (Fig. 2 in paper) has these components:

- **Fixed codebook ROM**: 30 bytes ($b{=}3$), shared across all $d$ dimensions
- **Sign vector ROM/eFuse**: 16 bytes/layer ($\mathbf{s} \in \{-1, +1\}^{d}$)
- **Write path circuit**: FWHT butterfly network ($\log\_{2}d$ stages, $d/2$ add/sub per stage = 448 add/sub for $d{=}128$), zero multipliers; Lloyd-Max comparator quantization (896 comparators)
- **Read path circuit**: Pre-computation table SRAM (2 KB), 128 parallel reads per key, adder tree (127 adders), norm scaling (1 multiplication per key)
- **KV cache storage**: 50 bytes/vector vs. 256 bytes FP16 = **5.1× compression**

## Training / Design

### Fixed Codebook Design

The codebook is computed offline using a Lloyd-Max solver with quantile-based initialization and `scipy.stats`, implemented independently (no code shared with prior work). For each $(d, b)$ pair, the solver produces centroids $\{c\_{0}, \ldots, c\_{2^{b}-1}\}$ and decision boundaries $\{b\_{1}, \ldots, b\_{2^{b}-1}\}$. These are verified against the Lloyd-Max optimality conditions to $< 10^{-11}$ precision. The codebook is model-independent and calibration-free.

### Sign Pattern Sensitivity Discovery

A major finding through 10-seed evaluation across 3 models (LLaMA-3.1-8B, Qwen2.5-3B, Qwen3-8B): **random sign patterns cause catastrophic PPL spikes** ($\Delta > 50$) on certain models. The root cause is layer-wise norm heterogeneity. Qwen2.5-3B layer 0 has mean key norm of 172 versus 22 at other layers (7.8× ratio), while LLaMA-3.1-8B shows only 1.5× ratio.

### Sign Pattern Optimization

A gradient-free selection algorithm (Algorithm 1) evaluates $C{=}200$ random sign candidates per layer on 8 calibration samples (WikiText-2 train), selecting the one minimizing quantization MSE. Cost: ~1 second/layer, ~36 seconds total for 36 layers. Storage overhead: $d$ bits/layer = 16 bytes/layer, 576 bytes total for 36 layers. The optimized signs occupy the **same ROM** as default signs --- zero additional hardware cost.

### Attention Score Computation

The read path computes attention in two phases:
1. **Phase 1** (once per query): Rotate query, generate pre-computation table ($d \times 2^{b}$ multiplications = 1024 for the default config)
2. **Phase 2** (repeated $T$ times): Per key: read $d$ indices from SRAM $\to$ table lookup $\to$ adder tree ($d - 1$ additions) $\to$ norm scaling (1 multiplication)

## Evaluation

### Multiplication Reduction

For $d{=}128$, $b{=}3$, $T{=}4096$:

$$\underbrace{d \cdot 2^{b}}\_{\text{table}} + \underbrace{T \cdot 1}\_{\text{norms}} = 1024 + 4096 = 5{,}120$$

versus conventional $T \cdot d = 524{,}288$: a **102.4× reduction**. This is a mathematical identity, not an approximation.

### Memory Savings

- KV cache: ~100 MB (AXELRAM 3-bit) vs. 512 MB (FP16) for 8B model, 32 layers, 8 KV heads, $T{=}4096$
- Fixed codebook ROM: 30 bytes
- Sign vector ROM: 16 bytes/layer
- Pre-computation table: 2 KB

### Perplexity Results (WikiText-2, 4-bit NF4 weights)

**Sign pattern sensitivity (Table I) --- PPL increase $\Delta$ over FP16 baseline:**

| Model | 2-bit (mean±std) | 2-bit (worst) | 3-bit (mean±std) | 3-bit (worst) | 4-bit (mean±std) | 4-bit (worst) |
|-------|-----------------|--------------|-----------------|--------------|-----------------|--------------|
| LLaMA-3.1-8B (FP16: 6.98) | +0.10±0.01 | +0.11 | +0.02±0.00 | +0.02 | +0.00±0.00 | +0.01 |
| Qwen2.5-3B (FP16: 8.29) | **+8.13±16.9** | **+58.43** | **+9.64±14.9** | **+51.00** | +1.10±0.85 | +2.98 |
| Qwen3-8B (FP16: 9.07) | +12.2±2.51 | +17.45 | +0.48±0.29 | +1.09 | +0.04±0.01 | +0.05 |

**After sign pattern optimization (Table III):**

| Model | 2-bit (worst→optimized) | 3-bit (worst→optimized) | 4-bit (worst→optimized) |
|-------|------------------------|------------------------|------------------------|

| Qwen2.5-3B | **+58.43 → +0.82 (99% reduction)** | **+51.00 → +0.58 (99% reduction)** | +2.98 → +0.25 (92%) |

### Key Observations

- LLaMA-3.1-8B is **fully stable** across all seeds and bit-widths (std $< 0.01$) --- correlates with low norm heterogeneity ratio ($\sim 1.5\times$)
- Qwen2.5-3B shows **catastrophic** variance (std $> 14$) from high norm heterogeneity ($7.8\times$), fully fixed by sign pattern optimization
- Qwen3-8B at 2-bit remains at $\Delta{=}{+}9.26$ even after optimization --- this is a fundamental 2-bit precision limit, not a sign pattern issue (best random seed gives +9.10)
- 3-bit provides excellent results on all 3 models tested; 4-bit is near-lossless
- QJL (1-bit residual correction from TurboQuant) is **harmful** at $b \leq 3$: the bit-stealing penalty outweighs bias correction because $\mathbb{E}[\exp(X)] \geq \exp(\mathbb{E}[X])$ by Jensen's inequality --- unbiased inner-product estimates yield biased (overestimated) attention weights after exponentiation

### Hardware Status

**No hardware has been fabricated.** All results are from bit-accurate software simulation in PyTorch. The 102.4× multiplication reduction is a mathematical identity that holds in any implementation. The architecture is presented as a proposal; silicon validation is future work.

## Reproduction Guide

```bash
# Clone the repository
git clone https://github.com/Axelidea/AXELRAM.git
cd AXELRAM

# Install dependencies
pip install -r requirements.txt  # torch, transformers, scipy

# Run sign pattern optimization (if needed for your model)
# Uses 200 candidates, 8 calibration samples from WikiText-2 train
# Takes ~1 second/layer, ~36 seconds for 36 layers
python optimize_signs.py --model meta-llama/Llama-3.1-8B-Instruct --bits 3

# Run perplexity evaluation
python evaluate.py --model meta-llama/Llama-3.1-8B-Instruct --bits 3 --dataset wikitext2

# Check layer-wise norm ratio to predict sign pattern sensitivity:
#   ratio < 2×: calibration-free (default signs) is safe
#   ratio > 5×: sign pattern optimization is recommended
python check_norm_ratio.py --model your-model-path
```

All evaluation code, codebook solver, sign pattern optimizer, and result JSON files are released under MIT license. The codebook solver uses quantile-based initialization and `scipy.stats`, independently implemented.

## Notes

- **This is hardware-level work** directly relevant to running LLMs on tiny/embedded devices. The entire point is avoiding dequantization at the circuit level --- the read path has no inverse transform.
- **Connects to [integer-softmax-approximation-edge-inference.md](/bopi-research/notes/integer-softmax-approximation-edge-inference.md):** AXELRAM's table-lookup attention combined with integer-only softmax approximations would enable a fully integer-arithmetic attention path on edge hardware.
- **The "compute from quantized representation without dequant" principle** is powerful for constrained devices. The asymmetric path (transform on write, lookup on read) is a general architectural pattern that could apply beyond attention to other dot-product-heavy operations.
- **Connects to [hadamard-rotation-quantization-dominance.md](/bopi-research/notes/hadamard-rotation-quantization-dominance.md):** This paper provides a concrete hardware realization of why Hadamard rotation dominates for quantization --- the $\mathcal{N}(0, 1/d)$ concentration enables a fixed codebook, which in turn enables table-lookup computation without dequantization.
- **Sign pattern sensitivity is a real deployment risk** that single-seed evaluations miss. The paper's 10-seed protocol should be standard practice for any rotation-based quantization method.
- **The 2-bit limitation** on Qwen3-8B ($\Delta{=}{+}9.26$) is inherent to 2-bit scalar quantization, not AXELRAM-specific. This suggests 3-bit is the practical floor for models with heterogeneous layer norms.
- **No silicon yet** --- this is an architecture proposal validated through simulation. The 102.4× multiplication reduction is exact (mathematical identity), but actual energy/latency savings depend on implementation technology.
- **Relates to [can-integer-only-neural-operations-enable-practical-foundation-model-inference-on-edge-hardware.md](/bopi-research/can-integer-only-neural-operations-enable-practical-foundation-model-inference-on-edge-hardware.md):** AXELRAM provides a concrete path for integer-only attention computation (3-bit indices + table lookups + adds + 1 FP16 multiply for norm), fitting the integer-only edge inference thesis.
