## problem
open-loop end-to-end neural motion planners for robotic manipulators generate a single path per workspace and don't leverage the open-loop structure for inference-time optimization. existing sampling-based planners (RRT, BIT*) are probabilistically complete but slow in complex environments. neural planners like MPNet and Motion Planning Diffusion (MPD) learn to plan but lack stochastic sampling at inference time for collision rejection. the core gap: no neural motion planner exploits the stochastic generative formulation to do best-of-$N$ sampling with a cheap collision checker at inference time.

## architecture
transformer-based encoder conditions a flow matching head. inputs: workspace point cloud $\mathcal{P}\_w$, robot point cloud $\mathcal{P}\_r$, current configuration $\mathbf{q}\_t \in \mathbb{R}^6$, goal configuration $\mathbf{q}\_{\text{goal}} \in \mathbb{R}^6$. point clouds are downsampled and embedded via PointNet++ encoders with output dimension $d=256$. configurations are embedded via a shared MLP. all embeddings are concatenated and processed by a transformer encoder (4 layers, 8 heads, $d=256$).

the flow matching head models the conditional path distribution $p\_0(\mathbf{x}^0)$ where $\mathbf{x}^0 = \delta\mathbf{q}\_{t:t+H-1}$ is the configuration sequence over horizon $H$. the optimal transport probability path interpolates:

$$x^\tau = (1-\tau)x^0 + \tau\epsilon, \quad \tau \in [0,1]$$

the conditional generating vector field:

$$u^\tau(x^\tau) = \mathbb{E}\_{p(x^0 \mid x^\tau)}[u^\tau(x^\tau \mid x^0)] = x^0 - x^\tau$$

a denoiser $v\_\theta$ is trained with the conditional flow matching loss:

$$\mathcal{L}\_{CFM} = \mathbb{E}\_{\tau, x^0, \epsilon} \left\| v\_\theta(x^\tau, \mathbf{c}) - (x^0 - x^\tau) \right\|^2$$

at inference time, $N$ candidate paths are sampled via Euler solver (20 steps), each evaluated by a collision checker with cost function $\mathcal{C}(q\_i)$, and the first collision-free path is selected.

## training
data generated by cuRobo (optimization-based planner) in LLM-generated environments. trained on NVIDIA RTX 4080. pytorch implementation. AdamW optimizer. 20 denoising steps for training.

## evaluation
compared against Bi-RRT, BIT*, MPNet, MPD (diffusion motion policy), and neural motion planner variants. evaluated on held-out planning tasks from prior work. FMP consistently outperforms both sampling-based and neural planners with and without inference-time optimization (best-of-$N=100$). key result: FMP with $N=1$ already matches or exceeds baselines, and best-of-100 pushes success rate substantially higher while maintaining planning time.

## reproduction guide
1. clone repo (code link pending in paper)
2. install pytorch, pointnet2 dependencies
3. generate training data via cuRobo in LLM-generated environments
4. train with default hyperparameters (4-layer transformer, $d=256$, 8 heads)
5. evaluate with euler solver, 20 steps, best-of-$N$ inference

## notes
the inference-time optimization via best-of-$N$ sampling is a clean pattern that separates the generative model from the physical constraint checker. this parallels the WAV paper's approach of using cheap verification to improve planning. the flow matching formulation is notably simpler than the diffusion alternative (MPD) while matching or exceeding performance. the architecture is relatively lightweight (4-layer transformer) compared to VLA-scale models, making it practical for real-time deployment. the key insight is that stochastic generative models for planning should explicitly exploit their sampling structure at inference time rather than outputting a single deterministic path.
